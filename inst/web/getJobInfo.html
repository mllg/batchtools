Query Job Information

getJobInfo(ids = NULL, pars.as.cols = FALSE, prefix.pars = FALSE, resources.as.cols = FALSE, reg = getDefaultRegistry())


getJobStatus(ids = NULL, reg = getDefaultRegistry())


getJobResources(ids = NULL, resources.as.cols = FALSE, reg = getDefaultRegistry())


getJobPars(ids = NULL, pars.as.cols = FALSE, prefix.pars = FALSE, reg = getDefaultRegistry())

Arguments

ids
[data.frame or integer] A data.frame (or data.table) with a column named “job.id”. This is usually the result of a findJobs operation, but you may also pass any other table here, e.g. the return of getJobInfo. Alternatively, you may also pass a vector of integerish job ids. If not set, defaults to all jobs.
pars.as.cols
[logical(1)] Transform the job parameters to data frame columns? Default is FALSE.
prefix.pars
[logical(1)] If set to TRUE, the prefix “par.” is used for a regular Registry to prefix parameter names of jobs while the prefixes “prob.par.” and “algo.par.” are used for an ExperimentRegistry. Has no effect if pars.as.cols is FALSE.
resources.as.cols
[logical(1)] Transform the resources data frame columns? Default is FALSE.
reg
[Registry] Registry.

Value

[data.frame] with the following columns in no particular order:
job.id
Unique Job ID as integer.

submitted
Time the job was submitted to the batch system as POSIXct.

started
Time the job was started on the batch system as POSIXct.

done
Time the job terminated (successfully or with an error) as POSIXct.

error
Either NA if the job terminated successfully or the error message.

time.queued
Time in seconds (as difftime) the job was queued.

time.running
Time in seconds (as difftime) the job was running.

memory
Estimate of the memory usage.

batch.id
Batch ID as reported by the scheduler.

job.hash
Unique string identifying the job or chunk.

resources
List of computational resources set for this job.

resources.hash
MD5 hash of the resource list.

pars
List of parameters/arguments for this job.

pars.hash
MD5 hash of the job parameters.

problem
Only for ExperimentRegistry: the problem identifier.

algorithm
Only for ExperimentRegistry: the algorithm identifier.

Description

getJobStatus returns the internal table which stores information about the computational status of jobs, getJobResources a table with the resources which were set to submit the jobs and getJobPars a table with the job parameters.

getJobInfo returns all these tables joined.
