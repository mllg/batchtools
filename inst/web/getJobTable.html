Arguments

ids
[data.frame or integer] A data.frame (or data.table) with a column named “job.id”. Alternatively, you may also pass a vector of integerish job ids. If not set, defaults to all jobs.
pars.as.cols
[logical(1)] Transform the job parameters to data frame columns? Defaults to TRUE if all parameters are atomic vectors, FALSE otherwise.
resources.as.cols
[logical(1)] Transform the resources data frame columns? Default is FALSE.
prefix
[logical(1)] If set to TRUE (default), the prefix “par.” is used to name column names of parameters for a Registry and prefixes “prob.par.” and “algo.par.” are used to name the columns of a ExperimentRegistry. Resources are prefixed with “res.”.
reg
[Registry] Registry.

Value

[data.frame] with the following columns in no particular order:
job.id
Unique Job ID as integer.

submitted
Time the job was submitted to the batch system as POSIXct.

started
Time the job was started on the batch system as POSIXct.

done
Time the job terminated (successfully or with an error) as POSIXct.

error
Either NA if the job terminated successfully or the error message.

time.queued
Time in seconds (as difftime) the job was queued.

time.running
Time in seconds (as difftime) the job was running.

memory
Estimate of the memory usage.

batch.id
Batch ID as reported by the scheduler.

job.hash
Unique string identifying the job or chunk.

resources
List of computational resources set for this job.

pars
List of parameters/arguments for this job.

pars.hash
MD5 hash of the job parameters.

problem
Only for ExperimentRegistry: the problem identifier.

algorithm
Only for ExperimentRegistry: the algorithm identifier.

Description

getJobStatus returns the internal table which stores information about the computational status of jobs, getJobResources a table with the resources which were set to submit the jobs and getJobPars a table with the job parameters.

getJobTable returns all these tables joined.
