ClusterFunctions for Local Multicore Execution

Usage

makeClusterFunctionsMulticore(ncpus = max(getOption("mc.cores", parallel::detectCores()), 1L), max.load = Inf, debug = FALSE)

Arguments

ncpus
[integer(1)] Number of VPUs of worker. Default is to use all cores but one, where total number of cores "available" is given by option mc.cores and defaults to the heuristic implemented in detectCores.
max.load
[numeric(1)] Load average (of the last 5 min) at which the worker is considered occupied, so that no job can be submitted. Default is Inf.
debug
[logical(1)] Set to TRUE to get more messages for debugging.

Value

[ClusterFunctions].

Description

Jobs are spawned by starting multiple R sessions on the command line.

See also

Other clusterFunctions: makeClusterFunctionsInteractive, makeClusterFunctionsSSH
