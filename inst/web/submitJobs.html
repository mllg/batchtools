Submit jobs to the Batch Systems

Usage

submitJobs(ids = NULL, resources = list(), reg = getDefaultRegistry())

Arguments

ids
[data.frame or integer] A data.frame (or data.table) with a column named “job.id”. This is usually the result of a findJobs operation, but you may also pass any other table here, e.g. the return of getJobInfo. Alternatively, you may also pass a vector of integerish job ids. If not set, defaults to result of findNotSubmitted.
resources
[named list] Computational resources for the batch jobs. The elements of this list (e.g. something like “walltime” or “nodes”) depend on by your template file, with the exception of ncpus and measure.memory. The setting ncpus is used to determine the number of CPUs to execute jobs in a chunk in parallel using mclapply. If not set, ncpus defaults to 1 (sequential execution). The second resource, measure.memory, can be set to TRUE to enable the measure of memory requirements using gc. But note that the reported values are quite heuristic and may not reflect the real memory requirements. Defaults can be set in the Registry via default.resources. Settings in resources overrule those in default.resources.
reg
[Registry] Registry. If not explicitly passed, uses the last created registry.

Value

[data.table]. Table with columns “job.id” and “chunk”.

Description

Submits all jobs to the batch system using the resources provided via resources.

If an additional column “chunk” is present in the table ids, the jobs will be grouped accordingly. See chunkIds for more information.
