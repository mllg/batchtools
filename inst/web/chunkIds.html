Chunk Jobs for Sequential Execution

Usage

chunkIds(ids = NULL, n.chunks = NULL, chunk.size = NULL, group.by = character(0L), reg = getDefaultRegistry())

Arguments

ids
[data.frame or integer] A data.frame (or data.table) with a column named “job.id”. This is usually the result of a findJobs operation, but you may also pass any other table here, e.g. the return of getJobInfo. Alternatively, you may also pass a vector of integerish job ids. If not set, defaults to all jobs.
n.chunks
[integer(1)] Requested number of chunks. If more chunks than elements in ids are requested, empty chunks are dropped. Cannot be used in combination with chunks.size.
chunk.size
[integer(1)] Requested number of elements in each chunk. Cannot be used in combination with n.chunks. If ids cannot be chunked evenly, some chunks will have less elements.
group.by
[character(0)] If ids is a data.frame with additional columns (besides “job.id”), then the chunking is done in subgroups defined by the columns group.by. See example.
reg
[Registry] Registry. If not explicitly passed, uses the last created registry.

Value

[data.table]. Table ids with additional column “job.id”.

Description

Partition jobs into “chunks” which will be executed together on the nodes.

Chunks are submitted via submitJobs by simply providing job ids as a table with an additional column “chunk”. All jobs with the same chunk number will be grouped together on one node as a single computational job.

Examples

# chunking for Registry
reg = makeTempRegistry(make.default = FALSE)
ids = batchMap(identity, 1:25, reg = reg)


Adding 25 jobs ...

ids = chunkIds(ids, chunk.size = 10, reg = reg)
print(ids)
