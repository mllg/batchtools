Computational Jobs

Usage

makeJob(id, reg = getDefaultRegistry())

Arguments

id
[integer(1) or data.table] Single integer to specify the job or a data.table with column job.id and exactly one row.
reg
[Registry] Registry. If not explictitly passed, uses the last created registry.

Value

[Job].

Description

makeJob takes a single job id and creates an object of class “Job”. This job is passed to reduce functions like reduceResults. Furthermore, it can be used for an ExperimentRegistry while in the user functions of each Problem and Algorithm. It is implemented as a list holding the following variables:

job.id
Job ID as integer.

pars
Job parameters as named list. For ExperimentRegistry, the parameters are divided into the sublists “prob.pars” and “algo.pars”.

seed
Seed which is set via doJobCollection.

resources
Computational resources which were set for this job.

Additionally, a regular Registry also has the slot “fun” with the user function while an ExperimentRegistry has the slots “problem” and “algorithm” (see Problem and Algorithm).

Examples

reg = makeTempRegistry(make.default = FALSE)


Sourcing configuration file '~/.batchtools.conf.r' ...

batchMap(identity, 1:5, reg = reg)


Adding 5 jobs ...

job = makeJob(1, reg = reg)
names(job)
