Run Jobs Interactively

Usage

testJob(id, fresh.session = FALSE, reg = getDefaultRegistry())

Arguments

id
[integer(1) or data.table] Single integer to specify the job or a data.table with column job.id and exactly one row.
fresh.session
[logical(1)] Run the job in a fresh R session? If TRUE, starts a fresh R session on the local machine to execute the with execJob. You will not be able to use debug tools like traceback or browser. If fresh.session is set to FALSE (default), testJob will execute the job in the current R session and you are unable to spot missing variable declarations (possibly resolved in the global environment) as well as forgotten package dependencies.
reg
[Registry] Registry. If not explicitly passed, uses the last created registry.

Value

[ANY]. Returns the result of the job if successful.

Description

Starts a single job in the current R session. Debugging tools like traceback should work.

Examples

reg = makeTempRegistry(make.default = FALSE)


Sourcing configuration file '~/.batchtools.conf.r' ...

batchMap(function(x) if (x == 2) xxx else x, 1:2, reg = reg)


Adding 2 jobs ...

testJob(1, reg = reg)
